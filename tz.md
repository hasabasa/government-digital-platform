ТЗ на переработку проекта: Из "Government Platform" в "Cube Business OS"
1. Общая задача
Полный рефакторинг существующего монорепозитория. Необходимо очистить проект от логики «государственной платформы» и адаптировать его под внутреннюю систему управления компанией Cube Demper.

2. Инфраструктурные изменения (По результатам аудита)
Docker-compose: Исправить docker-compose.yml. Включить в него все существующие сервисы: task-service, call-service и новый finance-service.

Унификация тестов: Полностью удалить jest из auth-service. Перевести все сервисы (backend и frontend) на Vitest.

Документация: Полное обновление README.md. Удалить описание госслужб, заменить на документацию по CRM и фин-модулю.

3. Рефакторинг Backend-сервисов
3.1. Очистка и удаление
Удалить disciplinary-service (дисциплинарные взыскания).

Удалить из user-service все поля, связанные с гос. должностями, ИИН (если не требуется) и прочей госслужбой.

Оставить и доработать: api-gateway, auth-service, user-service, chat-service, file-service.

3.2. Новый сервис: finance-service (Priority: High)
Создать микросервис для управления прибылью и долями.

Бизнес-логика (Расчеты):

Вход: Количество продаж по тарифам (21 990тг, 27 990тг, 33 990тг).

Авто-вычеты: Банк (2%), Налог (4%), Амбассадор Арслан (20% от каждой продажи).

Распределение чистого остатка: Хасенхан (40%), Адиль (40%), Азамат (20%).

Хранение: История всех транзакций и текущий баланс каждого дольщика в БД (PostgreSQL через Drizzle).

3.3. Доработка task-service и notification-service
Оживить task-service, связав его с фронтендом.

Реализовать отсутствующий notification-service для отправки уведомлений о новых задачах и финансовых поступлениях (WebSocket/Redis).

3.4. Оживление call-service
Интеграция с Google Meet API.

Создание ссылки на созвон при инициации звонка в чате.

4. Рефакторинг Frontend (apps/web)
4.1. Смена парадигмы интерфейса (Telegram Clone)
Полный редизайн Layout под стиль Telegram Desktop.

Левая панель: Список чатов/задач с группировкой по папкам («Разработка», «Маркетинг», «Продажи», «Финансы»).

Центральная часть: Окно чата, которое меняет функционал в зависимости от контекста (обсуждение задачи или ввод фин. данных).

UI-кит: Использовать темную тему (цвета VS Code / Cursor IDE), как предпочитает основатель.

4.2. Фин. интерфейс
Страница «Касса», где в 1 клик вводится количество продаж.

Дашборд для Хасенхана, Адиля и Азамата с графиком прибыли.

5. Стек и требования к коду
Frameworks: React (Vite), Node.js (TypeScript).

ORM: Drizzle ORM (оставляем и расширяем схемы).

Validation: Строго Zod для всех API-запросов, особенно в финансах.

State Management: Zustand (уже есть, использовать для управления состоянием чатов и балансов).

Ожидаемый результат:
Проект должен запускаться одной командой docker-compose up, иметь единую систему тестирования и полностью соответствовать логике распределения прибыли 40/40/20 после всех вычетов.
